Adaptive Baseline Calibration System and Premium Face Mesh Visualisation
A liveness verification specification document was produced covering thirteen sections with ten architecture decision records, state machine diagrams, detection algorithm documentation, and a sixteen-item verification plan. The most architecturally significant implementation introduced an adaptive baseline calibration system replacing fixed detection thresholds with per-user statistical measurements. A new calibrating state was inserted into the detection flow between face detection and active challenges, during which approximately forty frames of resting face metrics are accumulated to compute per-user mean and standard deviation for eye aspect ratio, head yaw angle, and smile ratio, with minimum standard deviation floors preventing impossibly tight thresholds. Quality gating validates calibration stability and retries up to three times before falling back to fixed thresholds. The three detection state machine functions were extended to accept an optional calibration baseline parameter: blink detection uses a 2.5-sigma closed threshold with absolute minimum floors, head turn detection centres around the calibrated resting yaw position with four-sigma turn detection, and smile detection uses a 2.5-sigma threshold above the resting smile ratio. A new Canvas 2D face mesh overlay component replaced the static CSS oval with a ten-layer real-time render pipeline comprising a horizontal scan sweep animation during calibration, sparse internal mesh connections with depth-modulated opacity, a thirty-six connection jawline and forehead face oval, eyebrow connections, sixteen-connection eye contours that pulse during blink challenges and turn green on completion, iris rings with radial gradient glow, forty-connection lip contours responsive to smile challenges, strategic key-point dots, a challenge completion flash ripple, and a circular calibration progress arc. The overlay uses device-pixel-ratio-aware canvas sizing via a resize observer for crisp rendering on high-density displays with all same-colour edges batched in single path-and-stroke calls for sub-millisecond frame rendering.

Real-Camera Threshold Tuning and Critical Fix Resolution
Real-camera testing with an actual webcam immediately revealed four problems that unit tests cannot detect. The sixty-frame calibration with strict quality thresholds was unrealistic for natural sitting posture as micro-head-movements easily exceeded the yaw standard deviation threshold, causing calibration to fail every attempt and fall back to fixed thresholds after nine seconds. The calibration frame count was reduced from sixty to forty and quality thresholds were relaxed to accommodate natural posture variation. Head turn detection was passing on slight weight shifts because the yaw threshold of 0.03 with only two required sustained frames detected noise rather than deliberate movement, so the threshold was raised to 0.05 with six sustained frames required for approximately three hundred milliseconds of deliberate rotation. Blink detection was nearly impossible because the adaptive threshold formula produced closed-eye thresholds below what some face geometries can physically achieve, prompting a three-pronged fix raising the fixed depth threshold, reducing adaptive sigma multipliers, and introducing absolute minimum floors ensuring no face geometry is required to produce an eye aspect ratio below 0.20. A duplicate calibration frame count constant defined independently in two files was consolidated to a single exported source of truth. A hysteresis floor was added to the adaptive blink detection ensuring a minimum 0.03 gap between the closed and open detection thresholds to prevent state machine deadlock. A null guard was added to the face mesh hook after two asynchronous operations where the video element reference could become null on component unmount, with proper media stream track cleanup. The test suite was expanded from sixty-five to ninety-seven tests with comprehensive coverage for six previously untested core utility functions and test helper factories generating realistic four-hundred-seventy-eight point landmark arrays.

Dashboard Recovery, Balance Formatting, and Anti-Spoofing Hardening
The dashboard page was returning a server error traced to the wallet service where a database query referenced a column added in a recent migration that had not been applied to the development environment. The migration was deployed after resolving a prerequisite stale migration referencing a renamed table and the service was restarted to restore functionality. Three separate currency formatting functions were unified to follow a single convention: whole numbers display without decimal places, fractional amounts show up to six decimal places with trailing zeros trimmed, and zero displays cleanly. A unit toggle was added to the dashboard balance card allowing users to switch between the primary currency display and its base unit denomination. The liveness detection system received anti-spoofing hardening against a hand-occlusion bypass where covering the face during active challenges caused blink and head turn to pass incorrectly. The face detection function was enhanced with geometric consistency checks validating inter-pupillary distance range, nose-below-eyes positioning, and face width proportionality. A new frame reliability function was introduced checking skeletal landmark displacement and eye aspect ratio rate-of-change between consecutive frames, with unreliable frames still updating visual metrics but skipping state machine updates. Detection thresholds were further tuned based on user feedback, making blink easier and head turn harder.
