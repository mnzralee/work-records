PROBLEMS ENCOUNTERED
A persistent wallet balance corruption bug was traced to a dual-write anti-pattern where the outbox-submitter worker's syncWalletBalanceFromBlockchain function wrote raw smallest-unit values directly to a standard-denomination database column without performing unit conversion, while a projector simultaneously applied correct relative updates to the same column. A systemic RBAC API response wrapping bug was discovered during end-to-end Playwright testing of the admin promotion flow, where all eight RBAC fetch functions returned the full API response wrapper instead of the inner data payload, causing permission counts to display as zero and checkboxes to fail to render. The OAuth2 PKCE SSO flow from Hub to administration portal required resolving five cascading layers of issues including an HTTP method mismatch, expired access tokens from a raw cookie reader that did not trigger JWT refresh, and a database schema drift where a NOT NULL constraint on a plaintext authorisation code column conflicted with the hash-only storage design. A monolithic 427-line admin creation page triggered six ESLint errors and twenty-eight warnings under React Compiler strictness rules, with successive fix rounds exposing deeper architectural issues including memoisation dependency instability from optional chaining, setState-in-effect anti-patterns, and useRef access violations during render. A Prisma schema drift caused by another developer's compliance work produced 500 errors on the contexts endpoint during E2E testing, and a critical security vulnerability was identified where transfers could proceed without PIN verification when the PIN feature was disabled.

SOLUTIONS FOUND
The balance corruption was resolved by adding the missing unit conversion division to both modular and legacy versions of the syncWalletBalanceFromBlockchain function. The systemic RBAC response wrapping bug was fixed by applying the response unwrapping pattern consistently across all eight fetch functions, which simultaneously repaired both the new promotion page and the existing roles management page. The OAuth2 SSO issues were resolved by switching the proxy from POST to GET with query parameters, replacing the raw cookie reader with a session handler that triggers JWT refresh, and executing a manual database column alteration with index removal to accommodate hash-only authorisation code storage. The monolithic admin page was progressively refactored over four rounds into a six-file component architecture comprising a 133-line orchestrator, four extracted components, and a custom permissions editor hook, with useEffect anti-patterns replaced by derived value patterns and render-time state adjustment. The Prisma schema drift was handled by adding error-tolerant handling to the institutional roles query, and the PIN bypass vulnerability was addressed by implementing a blocking state with redirect to security settings. Production readiness was further hardened through 16 fixes across 19 files including removal of a persisted plaintext OAuth authorisation code, CORS wildcard replacement with an explicit origin allowlist, and open redirect prevention via URL validation utilities.
