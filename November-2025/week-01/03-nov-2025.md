Monday, November 3rd, 2025 
Title: K8s Operational Tooling: Development of Automated Upgrade & Reset Scripts
Objective: To transition the newly migrated Kubernetes-based Fabric network from a static deployment to a fully operational and manageable system. This involved deprecating all legacy Docker scripts and creating a new suite of production-grade, K8s-native automation scripts for chaincode upgrades and full network resets, including authoring their comprehensive documentation.
Work Carried Out:
Script Repository Refactoring & Documentation: The day began by formally operationalizing the scripts/ directory for Kubernetes.
Deprecation: All 23 legacy scripts, which were designed for Docker Compose (e.g., deploy-prod.sh, backup.sh), were moved into a _deprecated/ subdirectory.
New README: A new scripts/README.md was authored to serve as the single source of truth. It clearly explains the new k8s-* script prefix, lists all deprecated scripts, and provides quick-reference kubectl commands for common operations.
Development of Chaincode Upgrade Tooling: A complete set of tools for automating chaincode upgrades in the K8s CCAAS (Chaincode-as-a-Service) environment was created.
Guide: Authored the comprehensive K8S_CHAINCODE_UPGRADE_GUIDE.md, detailing the full 8-step manual process, from building the image to the multi-organizational approval.
Automation Script: Based on the guide, the k8s-upgrade-chaincode.sh script was developed. This script automates the entire workflow: 1) building the new chaincode image, 2) pushing it to the K3s local registry, 3) creating the CCAAS package, 4) installing it on all peers, 5) updating the K8s chaincode Deployment, 6) approving for both organizations, 7) committing the definition, and 8) running a post-upgrade verification query.
Development of Network Reset Tooling: A critical (and dangerous) operational tool was needed to wipe the network for testing or recovery.
Guide: Authored the K8S_NETWORK_RESET_GUIDE.md, documenting the 10-step process and explicitly warning about the destructive nature of deleting Persistent Volume Claims (PVCs).
Automation Script: The k8s-reset-network.sh script was developed to automate this. It was built with extensive safety features, including multiple confirmation prompts, graceful scaling-down of all StatefulSets (Peers, Orderers, CouchDB), and the programmatic deletion and recreation of all 13 PVCs to ensure a completely clean state.
Iterative Debugging & Hardening of New Scripts: The newly created scripts required significant debugging to function in the live K8s environment.
Path Correction: A critical flaw was found: the scripts were using old Docker Compose file paths. A systematic audit was performed, and 15+ paths were corrected across both scripts to match the actual K8s pod filesystem (e.g., changing /var/hyperledger/fabric/organizations/.../admin/msp to the correct /tmp/admin-msp and /var/hyperledger/fabric/tls/ca.crt).
Upgrade Script Fixes: The k8s-upgrade-chaincode.sh script required more fixes. The K3s registry was found to be in the registry namespace (not kube-system). The sed command for updating package IDs was failing because the ID contains a colon (:); this was fixed by changing the sed delimiter from | to @.
Orderer/TLS Fixes: All orderer addresses were updated from external hostnames to the internal K8s service names (e.g., orderer0:7050) and the required --ordererTLSHostnameOverride flag was added to all peer commands.
Final Dry Run & Verification: The day concluded with a dry run of the fixed k8s-upgrade-chaincode.sh script.
Result: The script successfully upgraded the chaincode from v2.1 (seq 5) to v2.2 (seq 6), and post-upgrade queries worked, validating the script's core logic.
Known Issue: One minor bug was identified: a sed command (line 238) to update the CHAINCODE_ID in the deployment YAML failed during script execution but worked when run manually. This was documented as a known issue for future investigation.
Outcome: The K8s cluster is now equipped with a powerful, automated, and documented set of operational tools. The legacy Docker scripts are officially deprecated. New, production-ready scripts for fully automated chaincode upgrades (k8s-upgrade-chaincode.sh) and complete network resets (k8s-reset-network.sh) have been developed, debugged, and successfully tested, marking a major step in the operational readiness of the new infrastructure.
